## 0. 큰 그림 (프레이밍)

지금까지 구현한 건 **"경쟁 구조"**(레이스·라이벌·크루·대항전). 골격은 섰다.
다음에 필요한 건 그 위에 얹는 **"게임필(game feel) + 서사(narrative)" 레이어** — 매 판을 다르게 만들고, 라이벌을 캐릭터로 키우고, 결과를 밈으로 퍼뜨리는 것.

재활용 가능한 핵심 자산: 레이스 엔진·리더보드 · 라이벌 H2H 전적 · **`RankOvertake`(실시간 추월)** · **`path`의 t 타임스탬프** · 크루 대항전/잔디/명예의전당 · 안티치팅(실내런 승인) · 스토리카드/OG 렌더러 · Nudge 푸시.

점수 표기: **재미 / 신규성 / 적합(포지셔닝)** 각 1~10, 비용 S/M/L.

## 2. ⭐ 시그니처 베팅 — 고스트 레이스

지목한 라이벌(또는 크루원, 폴백=과거의 나)의 **그때 그 주행이 화면 위에서 실제 페이스대로 같이 달린다.** 이미 끝난 기록인데 눈앞에서 밀고 당기니 실시간 레이스급 긴장.

- **메커닉**: 기록 1건을 골라 포인트별 경과시간 `t`로 고스트 위치를 시간동기 재생 → 내 라이브 누적거리와 실시간 비교, 부호가 뒤집히면 "고스트 추월!"/추격 배너를 **기존 `RankOvertake` UI로** 발화. 경로가 달라도 순수 시간분면 비교라 공정.
- **왜 1순위(시그니처)**: (1) *"라이브로 시간 맞는 상대가 필요"*라는 코어 마찰을 유일하게 제거해 **모든 솔로런을 head-to-head로** 바꿈. (2) `path`의 t라는 우리만의 데이터로 RankOvertake 연출을 그대로 재활용. (3) 프라이버시 리스크 ≈ 0 (내·상대 과거 데이터) → 국가·연령 무관 즉시 확산. (4) 나머지 베팅의 인에이블러 — 라이벌 서사 아크는 잦은 재대결이 전제인데 그걸 고스트가 대량 생성.
- **MVP 첫 스텝**: 실시간·코스매칭 다 빼고 **"셀프 고스트 리플레이"**. 러닝 종료 후 내 직전 같은-거리 최고기록을 시간축 고스트로 재생 + "유령보다 12초 빨랐다"(성취 프레이밍) 카드. 유령을 *상대*로 인지하고 재도전하는지 검증 → 통하면 라이벌 고스트 → 실시간 순 확장.
- **재사용**: `path`의 t 보간, routeId 매칭, RankOvertake UI, WorkoutMap, 스토리카드.
