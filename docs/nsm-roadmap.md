# RunRace NSM 코치 로드맵

> 상태: 초안 · 작성 2026-06-30
> 관련: [roadmap.md](roadmap.md) (경쟁 축 — N3에서 P3 크루와 결합) · [product-positioning.md](product-positioning.md)
> 전제: NSM(노르웨이식 싱글 방법론)은 **런갤(러닝 마이너 갤러리) 수요 신호에서 출발한 훈련 코칭 버티컬**. 경쟁 축과 별개지만 N3에서 소셜로 합류한다.

---

## 배경

- 런갤에서 *"nsm 앱 만들어봐 / nsm 러닝프로그램 찾아봤는데 없더라"* 수요 신호 발견. 현재는 수기 *"딸깍 표"*(엑셀/표로 페이스·일정 계산)로 대체 중 — 제대로 된 앱 부재 = 빈 공간.
- NSM 코어는 구현 완료(N0). 다음은 **검증 → 개인 추적 → 소셜 확장** 순.

## 전체

| 단계 | 내용 | 성격 |
|---|---|---|
| **N0** | NSM 코어 (완료) | 빌드 완료 |
| **N1** | 검증 — 크루/런갤에 노출, 페이스·수요 확인 | **비-빌드, 게이트** |
| **N2** | 이행률 추적 — 세션 완료 기록 → 주간 진척 | 빌드 (개인 리텐션) |
| **N3** | NSM 소셜 — 크루·라이벌 NSM 도전 + 이행률 리더보드 | 빌드 (경쟁 P3 결합) |

### 순서 근거
- **N1(검증) 먼저**: 코어가 안 먹히면(페이스 틀림·수요 없음) 위 빌드가 통째로 헛심. 런갤 신호는 *가설*일 뿐 검증이 아님.
- **N2 전에 N1**: 까다로운 청중이라 페이스 정확도·실수요 확인이 투자 게이트.
- **N3는 N2 위에**: 이행률 데이터가 있어야 리더보드가 성립. 경쟁 로드맵 **P3(크루 레이스)** 와 합류점.

---

## N0 — NSM 코어 (완료, 2026-06-30)

구현:
- Daniels VDOT → 역치(Threshold) 페이스 (커뮤니티 표와 ±오차 일치 검증)
- **sub-T 요일 사용자 선택(2~3개)** → 주간 스케줄 결정적 생성 (특정 요일 강제 없음). `training_plan` (V35) + `sub_t_days` (V36), 사용자당 1개
- PB 자동 추천 칩(`/api/workouts/personal-bests`), 비로그인 계산 + 가입 유도, 홈 진입 버튼
- 런 중 오늘 sub-T 세션 자동 인식 + 렙별 목표/현재 페이스·휴식 카운트 (진행상태 localStorage 영속)
- i18n 5개 언어 (ko/en/es/ja/zh)

검증: tsc · vitest · next build · mvnw test/package · 다중 에이전트 교차검증(스케줄/번역)

---

## N1 — 검증 (비-빌드, 게이트)

목표: 코어에 더 투자할지 결정.
- **페이스 정확도**: 출력값을 런갤 *"딸깍 표"* / LacTrace / TheThresholdApp 과 ±2~3초/km 대조
- **수요**: YRC 크루·런갤 NSM러 3~5명에게 *"이 페이스 맞아? 이거 있으면 쓸래?"*
- **통과 기준**: 페이스 신뢰 + "쓰겠다" 신호 → N2 착수. 아니면 코어 보정(`THRESHOLD_VO2_FRACTION`·오프셋) 또는 중단.

---

## N2 — 이행률 추적 (빌드)

세션 완료를 기록해 개인 진척·리텐션을 만든다.
- 데이터: `nsm_session_log(plan_id, day/kind, workout_session_id, 적중률, completed_at)` 또는 `plan_session` 상태 컬럼
- 런 종료 시 NSM 세션이면 자동 로깅 → **"이번 주 sub-T 2/3 완료"** 표시
- 주간 리포트 + 다음 세션 안내
- 심박 가드는 health-sync 풀릴 때 (병렬 트랙, [health-sync-design.md](health-sync-design.md))

---

## N3 — NSM 소셜 (빌드, 경쟁 P3 결합)

크루/라이벌이 같이 NSM → 이행률 리더보드. 앱의 기존 소셜 강점(라이벌·레이스)과 결합.
- **크루 NSM 챌린지**: 같은 주간 sub-T 이행률 경쟁
- **라이벌 NSM**: 라이벌 세션 완료 알림(도발 푸시 인프라 재활용)
- 경쟁 로드맵 **P3(크루 레이스)** 와 합류 — 자동 집계가 단체전 마찰을 줄임

---

## 미결정 사항
- N2 로그 모델: 세션 단위 로그 vs 주간 집계
- N3가 경쟁 P3에 흡수되는가 vs 별도 "NSM 크루"
- 심박 가드 시점 (health-sync 심사 게이트)
- NSM이 별도 진입점(`/training`) 유지 vs 운동/레이스 탭에 통합
