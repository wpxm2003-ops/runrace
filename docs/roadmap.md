## 전체 로드맵

P0~P4는 **경쟁 기능 축**(서로 의존, 순차로 쌓임). 헬스 동기화는 그 줄에 끼는 기능이 아니라 **모든 단계에 공통으로 작용하는 입력 파이프라인(병렬 인에이블러)** — 보류 중, Play 건강데이터 심사가 게이트.

| 단계 | 내용 | 전략 버킷 |
|---|---|---|
| **P0** | 라이벌리 기반 — 승부 요약 + 전적 (읽기 토대) | 해자·경쟁깊이 |
| **P1** | 라이벌 레이스 — 재대결 + 라이벌리 스코어링 + 선별 알림 | 해자·경쟁깊이 |
| **P2** | 랭킹전 + 티어 | 리텐션 엔진 |
| **P3** | 크루 레이스 (단체전) | 성장·바이럴 |
| **P4** | 우승상품 | 수익화 |
| (병렬) | 헬스/워치 동기화 — 경쟁 연료 수집 | 인에이블러 (보류) |

### 순서 근거
- **P0 먼저**: 나머지 셋의 토대. 칼럼 1~2개로 싸게 시작.
- **P1을 P2 앞에**: 토대 위 가장 싼 감정 훅을 먼저 검증한 뒤 무거운 랭킹전 진입.
- **P2 랭킹전은 엔진이자 최대 리스크**: 점수 밸런싱이 게임 디자인 난제.
- **P3 성장 → P4 수익화**: 끈끈해진 베이스 위에서 크루가 사람을 끌고, 그 위에 수익화.

### 병렬 트랙 점화 시점
- P0~P1: 불필요 (유저 적음, 수동 입력 충분, 심사 부담만 큼) → 보류 유지
- **P2 전후가 적기**: 매일 뛰게 만드는 엔진이 생기면 자동 임포트가 참여율을 끌어올림. 심사 리드타임 역산해 P2 개발과 병행 착수.
- P3: 크루원 기록 자동 집계로 단체전 마찰 급감 → 효과 극대화

## P0 — 라이벌(팔로우식) + 라이벌 전적 (재구현 2026-06-12)

> 1차 구현(전원 전적 노출)은 벽지화 문제로 되돌렸고, **라이벌 모델로 재설계해 재구현**. 핵심 변경: 전적을 모두에게 반복 노출하지 않고 **내가 등록한 라이벌에게만** 노출(희소성 확보, [[result-screen-no-defeat-framing]]). 승부 요약은 "졌어요" 대신 **추격 프레이밍**("…까지 N초!").
>
> **구현 내용 (워킹트리, 커밋 안 함):**
> - 라이벌 = 팔로우식 단방향(요청·수락 없음). 테이블 `rival`(V23), 닉네임으로 등록/해제, 목록+전적(승/패/승률).
> - 내정보에 "라이벌 관리" 버튼 → `/rivals` 페이지(안내·등록 input·목록·해제).
> - `final_rank`(V22) self-join으로 전적 도출(집계 테이블 0). 종료 2경로에서 순위 확정/되돌림.
> - 레이스 상세: 라이벌 행 주황색 + "라이벌" 라벨(전 단계). 종료 시 라이벌 행에 전적 + 승부욕 추격 배너.
> - 검증: 백엔드 컴파일·기동(Flyway V22/V23 적용, Hibernate validate 통과)·프론트 타입체크·린트 통과. 로컬 시드 투입(전적 3승2패 + 예정/종료 showcase).
>
> **남은 것:** 사용자 로컬 화면 확인. (재대결·라이벌 자동감지/선별알림은 여전히 P1.)

### 한 줄 목표
> 레이스가 끝나면 "누가 누굴 이겼는지 + 나와의 전적"이 보이기 시작한다. (토대 + 읽기 전용)

### 포함
1. **승부 요약 화면** (레이스 종료 시)
   - 최종 순위 표시
   - 차이: **완주끼리 = 시간 차** (`finished_at` 차이) / **미완주 = 거리 차** (`goal - total_km`)
   - 스냅샷 없는 단순안 채택 (아래 "보류된 정교화 B" 참고)
2. **전적 표시**
   - **뷰어(나) 기준** — 참가자 전원 나열 X
   - **인접 순위 상대 중심**: 바로 위(놓친 상대)·아래(막아낸 상대) + "민수 2승 3패"
   - 나머지 참가자는 탭하면 1:1 전적 조회
   - 전부 `final_rank` 쿼리로 계산 → 저장 0, 모든 쌍 + 누적 히스토리 자동
3. **N명 전적 의미론 = 순위 기반** (확정)
   - 등수만 저장, 쌍 전적은 파생. 10명전이든 50명전이든 신규 테이블 0.

### 전적이 쿼리로 나오는 원리 (확정된 모델)
`challenge_member`에 사람당 `final_rank` 한 값만 저장. "A vs B 전적"은 둘 다 참가한 레이스들을 self-join 해 `final_rank`를 비교·합산.

```sql
-- 나(:me) vs 상대(:op) 누적 전적 — 둘 다 참가한 끝난 레이스 전부 합산
select
  sum(case when me.final_rank < op.final_rank then 1 else 0 end) as wins,
  sum(case when me.final_rank > op.final_rank then 1 else 0 end) as losses
from challenge_member me
join challenge_member op
  on op.challenge_id = me.challenge_id and op.user_id = :op
join challenge c on c.id = me.challenge_id and c.is_ended
where me.user_id = :me;
```
- "여러 번 진행한 누적 정보" = 행이 쌓일 뿐, 매번 합산이라 자동 반영
- 임의의 두 명 모두 조회 가능 (인접/비인접 무관)

### 데이터 변경 (작음)
- `challenge_member`에 **`final_rank`** 추가 (종료 시 확정)
- 신규 테이블 0개

### 제외 → P1 이후
- **재대결 일체** (사용자 결정: P1로 미룸)
- 라이벌 감지/스코어링, 선별 알림, 자동 라이벌 nag(스팸 방지)
- `head_to_head` 집계 테이블 (쿼리 느려질 때)
- "항상 거리 차" 스냅샷 (정교화 B)

## P1 — 라이벌 레이스

P0의 전적 데이터를 *연료*로 쓰는 능동적 콘텐츠. "네 라이벌을 알아보고, 꽂힐 때만 찌르는 시스템" — 삼성헬스가 구조적으로 못 하는 해자.

### 1. 재대결 (P0에서 이관 — 멀티플레이어 공정성 한 곳에 모음)
- **타깃 초대 단위**: 큰 레이스 재대결도 "전체 방 복제"가 아니라 **특정 상대(기본=인접 라이벌) 1명 초대**
- **비공개/초대 전용 방**: 공개 목록에서 제외 + 초대받은 사람만 join (난입 방지)
- **명단 고정**: 지정 상대만 입장
- **수락 시 동시 출발** (PENDING 상태): 둘 다 0에서 공정 출발. 과거 제거된 `status` 개념을 가벼운 형태로 복귀 필요 (`V5__challenge_drop_type_status` 참고)
- 목표·기간은 원본에서 상속. 원본이 기한 없는 목표전이면 리매치도 기한 없음.
- `challenge`에 `rematch_of_id` 추가 → 리매치 체이닝/스토리

### 2. 라이벌리 스코어링 (라이벌은 선언이 아니라 창발)
신호: **접전도**(박빙 차) · **전적 균형**(5승4패=라이벌 / 10승0패=학살, 제외) · **반복 대면** · **최근성** · **상호작용**(Nudge 주고받음). → 라이벌리 스코어.

### 3. 선별 알림 (희소성이 핵심)
- 버튼은 항상 수동 노출(P0), **푸시는 고신호 순간에만**: 결정적 접전 / 스트릭·기록 흔들림 / 이미 라이벌인 상대만
- 레이트리밋: 하루 1건, 상대당 쿨다운
- 매번 알리면 벽지가 됨 → 희귀해야 한 방이 꽂힘

### 4. (옵션) 승부 요약 정교화 B
- `km_at_winner_finish` 스냅샷: 승자 목표 도달 순간 나머지 거리 캡처 → "민수 결승 통과 시 0.3km 뒤"를 **항상 거리로** 표현. P0의 단순 분기(시간/거리) 대신.

## P2 — 랭킹전 + 티어

자유 러닝 → 페이스·거리 기반 랭크포인트 → 순위별 티어. 일일 습관 엔진(최강 리텐션).

### ⚠️ 설계 난제 (4단계 중 1위)
- **점수 공식 밸런싱**: 거리 지수 증가가 양날의 검 — 페이스 빠른 10k 주자보다 설렁설렁 100km가 이기면 망가짐. 페이스×거리 가중 + 상한/감쇠 곡선 필요.
- 시즌제(티어 리셋), 안티치팅(`IndoorRunApproval` 활용), 매칭 풀 크기, 스머핑 방지

---

## P3 — 크루 레이스 (단체전)

크루(팀) 엔티티 + 크루 단위 레이스. 성장·바이럴 벡터(크루가 사람을 끌어옴).

### ⚠️ 고려
- 크루 인원 차 공정성(인당 평균 or 핸디캡)
- 유령 멤버 처리

---

## P4 — 우승상품

레이스에 상품 걸기. 판돈이 감정 투자를 키우지만 리스크 최대 → 최후순위.

### ⚠️ 리스크
- **한국 사행성(도박) 규제**: 참가비+상금 구조는 규제 대상 가능 → **기프티콘 스샷 방식(돈 안 만짐) 먼저**, 결제/에스크로는 한참 뒤
- 배송 신뢰·세금·환불·어뷰징

## 미결정 사항
- **P1 ↔ P2 순서**: 추천은 P1(라이벌 레이스) 먼저. 단 "랭킹전 먼저 띄워 일일 리텐션 조기 확보"도 합리적(대신 밸런싱 리스크 조기).
- P2 점수 공식 구체안 (페이스×거리 가중·감쇠 곡선)
- P3 크루 공정성 방식 (인당 평균 vs 핸디캡)
- P4 상품 정산 방식 (기프티콘 스샷 → 결제 단계 진입 시점)
